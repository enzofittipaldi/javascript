Operador Módulo
	-O operador módulo % informa o que sobra após dividir um número por outro.
	result = dividend % divisor

	-dividend: O número que está sendo dividido.
	-divisor: O número que divide o dividendo.
	-result: O resto da divisão.

	-Por exemplo,
	-result = 10 % 3
	-Aqui, 10 é dividido por 3. 3 cabe em 10 três vezes, com um resto de 1. Então, result será 1.

	-Geralmente, o módulo é usado para verificar se um número é par ou ímpar:
	-Se um número é par, dividi-lo por 2 deixará um resto de 0.
	-Se um número é ímpar, dividi-lo por 2 deixará um resto de 1.

	-Quando o dividendo é menor que o divisor, ele não pode ser dividido, então o dividendo inteiro permanece.

Atalhos Aritméticos
	-O JavaScript criou um atalho legal para operações aritméticas de autoatribuição.

	-Por exemplo, em vez de escrever:
	let a = 5
	a = a + 3 // a holds 8

	-Podemos simplificá-lo escrevendo +=:
	let a = 5
	a += 3 // a holds 8

	-O += está adicionando ao a ele mesmo o valor 3

	-Esta operação é válida para todas as operações aritméticas:
	Operador	Atalho
	+		+=
	-		-=
	*		*=
	/		/=
	%		%=

Igualdade Estrita vs. Laxa



	- Em JavaScript, existem dois tipos de operadores de igualdade: igualdade frouxa (==) e igualdade estrita (===).

	- Igualdade frouxa (==) verifica se dois valores são iguais após realizar coerção de tipo. Isso significa que, se os operandos tiverem tipos diferentes, o JavaScript tentará convertê-los para um tipo comum antes de fazer a comparação.

Por exemplo:

	- let b = 5 == "5";   // É verdadeiro, porque a string "5" é convertida para o número 5 antes da comparação
	- let b = 0 == false; // É verdadeiro, porque o booleano false é convertido para o número 0 antes da comparação
	- let c = "" == false; // É verdadeiro, porque a string vazia é convertida para o número 0 antes da comparação

 Igualdade estrita (===), por outro lado, verifica se dois valores são iguais sem realizar qualquer conversão de tipo. Se os operandos tiverem tipos diferentes, eles são considerados não iguais.
Por exemplo:

	- let b = 5 === "5";   // É falso, porque nenhuma conversão de tipo é realizada, e um número não é igual a uma string
	- let b = 0 === false; // É falso, porque nenhuma conversão de tipo é realizada, e o número 0 não é igual ao booleano false

Em geral, é recomendado usar igualdade estrita (===) na maioria dos casos para evitar resultados inesperados devido à coerção de tipo.
Da mesma forma, também existem operadores de desigualdade frouxa (!=) e desigualdade estrita (!==) que funcionam de maneira semelhante, mas verificam desigualdade em vez de igualdade.
Desigualdade frouxa (!=) verifica se dois valores não são iguais após realizar coerção de tipo, enquanto desigualdade estrita (!==) verifica se dois valores não são iguais sem realizar conversão de tipo.

Operadores Lógicos Parte 1

Operadores lógicos são usados para verificar combinações de comparações que retornam true ou false.

Por exemplo, a seguinte declaração contém duas comparações: 

5 é maior que 3 e menor que 6?

Operador	Significado					Exemplo
&&		E - true se todos os operandos forem true	a && b
||		Ou - true se qualquer operando for true		a || b
!		Não - true se o operando for false		!a
 

Vamos ver alguns exemplos:

5 é maior que 3, e 1 é igual a 1:

let b1 = (5 > 3) && (1 == 1); // resulta em true
Explicação: Todos os operandos são true, então b1 conterá true (&& resulta em true se ambos os operandos forem true) .

 

5 não é igual a 4, ou 5 é igual a 2:

let b2 = !(5 == 4) || (5 == 2); // resulta em true
Explicação: O primeiro operando (5 != 4) é true, então b2 também é true (|| resulta em true se qualquer um dos operandos for true)

 

1 não é igual a 1 ou false:

let b3 = !(1 == 1) || false; // resulta em false
Explicação: Todos os operandos são false, então b3 conterá false (|| operação).

 

não (3 maior que 4):

let b4 = !(3 > 4); // resulta em true
Explicação: O operando é false, então b4 conterá true (! operação).

 

não (5 maior que 10 ou 5 maior que 1):

let b5 = !(5 > 10 || 5 > 1); // resulta em false
Explicação: 5 > 10 || 5 > 1 é true (um dos operandos é true), então no total b5 é false (! operação).

Operadores Lógicos Parte 2



Operadores lógicos têm uma tabela especial chamada "Tabela verdade" que mostra o que a combinação de operadores lógicos retorna.

Tabela verdade para o operador &&:

a	b	a && b
false	false	false
false	true	false
true	false	false
true	true	true
A única maneira de obter um true para o operador && é se ambos a e b forem true

Tabela verdade para o operador ||:

a	b	a || b
false	false	false
false	true	true
true	false	true
true	true	true
Neste caso, para obter um resultado true, a ou b deve ser true.

Tabela verdade para o operador !:

a	!a
false	true
true	false
Aqui o valor de a é invertido. Se a for false então !a é true

Ao trabalhar com expressões lógicas, às vezes precisamos simplificá-las ou rearranjá-las.

! (não) na frente de duas condições unidas por && (e), você pode dividi-la em duas partes separadas. O && (e) se torna || (ou), e cada parte recebe seu próprio ! (não):

!(A && B) é o mesmo que (!A) || (!B)

Por exemplo:

// Let's check if a number is NOT (between 1 and 10)
let number = 15

// These two expressions are equivalent:
let result1 = !(number >= 1 && number <= 10)
let result2 = !(number >= 1) || !(number <= 10)

console.log(result1)  // True
console.log(result2)  // True
O oposto também é correto: !(A || B) é o mesmo que (!A) && (!B)

Por exemplo:

// Checking if a person is NOT (a student or employed)
let is_student = false
let is_employed = false
// These two expressions are equivalent:
let result1 = !(is_student || is_employed)
let result2 = !is_student && !is_employed

console.log(result1)  // True
console.log(result2)  // True

Type Coercion



A coerção de tipo em JavaScript é a conversão automática de valores de um tipo de dados para outro. Isso pode acontecer de forma implícita em certas operações ou explicitamente quando você usa funções como String(), Number() ou Boolean().

A coerção de tipo implícita frequentemente ocorre quando você usa o operador de igualdade frouxa == ou quando realiza operações entre tipos diferentes, como adicionar um número a uma string.

Por exemplo:

let result = 5 + "5";
// Holds "55", because the number 5 is implicitly converted to the string "5" and then concatenated
let result = true == 1;
// Holds true, because the boolean true is implicitly converted to the number 1
Você pode converter valores manualmente com: (Coerção de tipo explícita)

String(value)
Number(value)
Boolean(value)
Number("123") + 7; // 130
Number(true);      // 1 
⚠️ Importante: Truthiness e Boolean()
Boolean() não interpreta o conteúdo da string como "true" ou "false".
Em vez disso, segue as regras truthy/falsy do JavaScript:

String vazia ("") → false
Qualquer string não vazia → true
Portanto:

Boolean("false"); // true  ← non-empty string
Boolean("");      // false 
Isso é uma fonte comum de confusão.

Estrutura If



Instruções if nos permitem executar código com condições.

Por exemplo, vamos olhar para o seguinte código:

let age = 20;
let status = "Child";
if (age > 18) {
    status = "Adult";
}
age += 1;
O código acima verifica se a variável age é maior que 18. Se for, ele definirá status para conter a string "Adult".

No final, o código incrementará age em 1, independentemente de a idade ser maior que 18 ou não.

 

Para usar uma instrução if, precisamos adicionar parênteses () que determinarão a condição, e tudo o que estiver dentro do if fica entre chaves {}:

if (condition) {
    code;
    code;
    code;
}
Se a condição for verdadeira, entraremos no bloco de código dentro do if (o código dentro das chaves)

If - Else



if nos permite executar um código específico se uma condição for atendida, mas e se quisermos executar outra coisa se a condição não for atendida?

Para isso, temos a instrução else:

let age = 15;
let status = "None";
if (age >= 18) {
    status = "Adult";
} else {
    status = "Young";
}
No exemplo acima, age é menor que 18, o que significa que ele entra no código else e status conterá "Young".

Podemos até torná-lo mais profundo usando a instrução else if:

let age = 68;
let status = "None";
if (age < 18) {
    status = "Young";
} else if (age >= 18 && age <= 65) {
    status = "Adult";
} else {
    status = "Old";
}
Aqui ele verifica se age é menor que 18, caso contrário, continuará para a próxima condição e verificará se age está entre 18 e 65. Se essa condição também não for atendida, ele definirá status como "Old".

Podemos adicionar quantas instruções else if quisermos:

if (condition1) {
    code;
} else if (condition2) {
    code;
} else if (condition3) {
    code;
    ...
